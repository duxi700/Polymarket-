## 可能的原因分析

根据 `config/thresholds.yaml` 中的 Level 3 信号触发条件：

```yaml
level3:
  # 核心条件：极端价格变化
  extreme_price_change_pct: 0.25  # 25% ✅ 已满足（多次达到25-36%）
  
  # 确认条件：回弹检测
  rebound_enabled: true
  rebound_check_window: 300  # 5分钟
  rebound_min_pct: 0.05  # 5% ❓ 可能未满足
  
  # 过滤条件
  min_liquidity: 1000  # 最小流动性 ❓ Mock数据可能不足
  
  # 冷却机制
  signal_cooldown_minutes: 15  # ✅ 首次信号无冷却限制
```

### 推测的触发失败原因：

1. **缺少回弹 (Rebound)**
   - 价格虽然大幅跳跃，但可能没有在5分钟窗口内出现5%以上的反向回弹
   - 例如：从0.50跌到0.20后，需要在300秒内回升到至少0.21才算有效回弹

2. **流动性不足**
   - Mock订单簿中的流动性（bid/ask size之和）可能小于1000
   - 当前Mock生成的size范围：100-1000随机，总和可能不够

3. **时间窗口不足**
   - 需要足够的历史数据点来计算1分钟/5分钟/15分钟的价格变化
   - 测试运行时间较短，可能缺少足够的时间序列数据
